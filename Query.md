
## to check which fk blong to whom without opening db

```
SELECT
    fk.name AS FK_Name,
    OBJECT_NAME(fkc.parent_object_id) AS ChildTable,
    cp.name AS FK_Column,
    OBJECT_NAME(fkc.referenced_object_id) AS ParentTable,
    cr.name AS PK_Column
FROM sys.foreign_key_columns fkc
INNER JOIN sys.foreign_keys fk
    ON fk.object_id = fkc.constraint_object_id
INNER JOIN sys.columns cp
    ON cp.object_id = fkc.parent_object_id
    AND cp.column_id = fkc.parent_column_id
INNER JOIN sys.columns cr
    ON cr.object_id = fkc.referenced_object_id
    AND cr.column_id = fkc.referenced_column_id
WHERE OBJECT_NAME(fkc.parent_object_id) = '<TABLE_NAME>';


```

### Square Brackets (`[]`) in SQL Server

Square brackets are used to explicitly identify database objects such as table names, column names, schemas, and databases.

Examples:

```sql
SELECT Name FROM Characters;
SELECT [Name] FROM [Characters];
```

Both queries are equivalent when the object names are simple and not reserved keywords.

Square brackets are required when:

* The name is a SQL keyword (`[Order]`, `[Group]`, etc.)
* The name contains spaces (`[First Name]`)
* The name contains special characters (`[User-Name]`)
* You want to ensure SQL Server treats the text as an identifier

**Best Practice:** Use meaningful object names such as `CharacterName`, `GameName`, and `CreatedDate` to avoid needing brackets in everyday queries. While brackets are safe and commonly generated by tools like SSMS, most developers use them only when necessary.
